# Plan de pruebas: Productos

## Rutas cubiertas
- `/home/products/varieties`
- `/home/products/formats`

## Casos sugeridos
1. **Variedades**: agregar una nueva variedad, editar sus atributos y eliminarla asegurando que la lista refleja los cambios.
2. **Formatos**: crear un formato de producto y preparar un reporte que utilice este formato, verificando consistencia del nombre y descripción.
3. **Filtros**: aplicar filtros por estado (activo/inactivo) y confirmar que los registros listados son los esperados.
